## 1. 총운 (yearSummary)

### 세운 기준
- **세운 천간**: 2026년 병(丙)
- **세운 지지**: 2026년 오(午)
- **주도 십성**: 일간 대비 세운 지지(午)의 십성 (`calculateSipsin(dayMaster, '午')`)
- **보조 십성**: 일간 대비 세운 천간(丙)의 십성 (`calculateSipsin(dayMaster, '丙')`)
- **이벤트**: 세운 지지(午)와 원국 지지의 관계 (충/합/형/파/해/없음)
- **용신/기신**: 세운 오행이 용신/기신 오행과 일치하는지 여부

### 점수 계산식
```
총운 점수 = 기본 75점
          + 십성 길흉도 (식신/정재/정관/정인: +10점)
          + 이벤트 반영 (충: -5점)
          + 오행 균형도 (부족한 오행 없음: +5점)
          + 용신/기신 (용신해: +10점, 기신해: -10점)
최종 범위: 45~95점
```

### 내용 생성 기준

#### 1-1. summaryText 생성
**조건 1**: 용신해인 경우
```
"[연도]년은 용신 [용신명]이 들어오는 해로, 전반적으로 운세가 상승합니다. 
[주도 십성]이 주도하는 해로, 내면에 잠들어 있던 목표의식이 현실화되는 역동적인 해입니다."
```

**조건 2**: 기신해인 경우
```
"[연도]년은 기신 [기신명]이 강한 해로, 신중한 처신이 필요합니다. 
[주도 십성]이 주도하는 해로, 변화에 대비하며 신중하게 나아가야 합니다."
```

**조건 3**: 그 외
```
"[주도 십성]이 주도하는 해로, 내면에 잠들어 있던 목표의식이 현실화되는 역동적인 해입니다."
```

**참고**: `josa()` 함수를 사용하여 조사 처리 (예: "이/가", "을/를")

#### 1-2. 점수 표기
- 점수는 45~95점 범위로 표시
- 점수에 따른 등급은 별도로 표기하지 않음 (UI에서 처리)

## 2. 올해 운의 성격 (yearNature)

### 세운 기준
- **이벤트**: 충/합/형/파/해/없음
- **주도 십성**: 비견/겁재/상관 등

### 계산식
```
pace 결정:
- 충이면 → 'fast'
- 주도 십성이 상관 또는 겁재면 → 'fast'
- 그 외 → 'slow'

quality 결정:
- 충 또는 형이면 → 'volatile'
- 합이면 → 'stable'
- 그 외 → 'mixed'
```

### 내용 생성 기준
```
pace === 'fast' ? '빠르고 ' : '차분하고 '
+ quality === 'volatile' ? '변동성이 큰 ' : '안정적인 '
+ '해'
```

**예시**:
- "빠르고 변동성이 큰 해"
- "차분하고 안정적인 해"
- "빠르고 안정적인 해"

## 3. 운의 작동 방식 (yearMechanism)

### 세운 기준
- **주도 십성**: 일간 대비 세운 지지(午)의 십성

### 매핑 규칙
| 주도 십성 | 운의 작동 방식 타입 | 설명 |
|-----------|-------------------|------|
| 비견, 겁재 | competitive-focused | 경쟁과 자기주장이 강해지는 해 |
| 식신, 상관 | creative-focused | 창의성과 표현력이 빛나는 해 |
| 편재, 정재 | wealth-focused | 재물과 사업이 중심이 되는 해 |
| 편관, 정관 | authority-focused | 책임과 권한이 커지는 해 |
| 편인, 정인 | thinking-first | 감정보다 판단과 분석이 먼저 작동하는 해 |

### 내용 생성 기준

#### 3-1. description
각 타입별 고정된 설명문 사용:
- `thinking-first`: "감정보다 판단과 분석이 먼저 작동하는 해"
- `action-first`: "생각보다 몸이 먼저 움직이는 해"
- `relationship-focused`: "인간관계가 운의 핵심이 되는 해"
- `wealth-focused`: "재물과 사업이 중심이 되는 해"
- `authority-focused`: "책임과 권한이 커지는 해"
- `creative-focused`: "창의성과 표현력이 빛나는 해"
- `competitive-focused`: "경쟁과 자기주장이 강해지는 해"

#### 3-2. advantage (강점)
각 타입별 고정된 배열 사용 (3개 항목):
- 예: `thinking-first` → ["분석력 향상", "기획력 강화", "지식 축적"]

#### 3-3. risk (위험)
각 타입별 고정된 배열 사용 (3개 항목):
- 예: `thinking-first` → ["결정 지연", "과도한 고민", "실행력 부족"]

## 4. 흔히 겪는 실수 패턴 (commonMistake)

### 세운 기준
- **주도 십성**: 운의 작동 방식 타입 결정
- **이벤트**: 충/합/형/파/해/없음

### 내용 생성 기준

#### 4-1. title
운의 작동 방식 타입별 고정된 제목 사용:
- `thinking-first`: "생각만 하다 타이밍을 놓치기 쉬운 해"
- `action-first`: "충동적으로 움직이다 후회하기 쉬운 해"
- `relationship-focused`: "타인에게 휘둘리기 쉬운 해"
- `wealth-focused`: "욕심에 눈이 멀어 균형을 잃기 쉬운 해"
- `authority-focused`: "완벽주의에 빠져 지치기 쉬운 해"
- `creative-focused`: "현실과 동떨어진 이상만 좇기 쉬운 해"
- `competitive-focused`: "불필요한 경쟁으로 적을 만들기 쉬운 해"

#### 4-2. situations (상황들)
**기본**: 운의 작동 방식 타입별 고정된 4개 상황

**이벤트별 추가 상황**:
- **충**: "주변 환경의 급격한 변화에 휘둘려 판단력을 잃음"
- **형**: "자기 파괴적인 패턴을 반복하며 스스로를 지치게 함"
- **합**: "좋은 기회를 앞에 두고도 망설이다 놓침"
- **파/해/없음**: 추가 상황 없음 (기본 4개만 사용)

**최종 배열**: 기본 3개 + 이벤트 추가 상황 1개 = 총 4개

## 5. 용신 분석 (yongshin)

### 세운 기준
- **용신 정보**: `sajuData.yongshin`에서 가져옴
- **세운 오행**: 세운 천간(丙)과 지지(午)의 오행
- **용신해 여부**: 세운 오행이 용신 오행과 일치하는지
- **기신해 여부**: 세운 오행이 기신 오행과 일치하는지

### 내용 생성 기준

#### 5-1. 용신 정보 표기
```
주 용신: [yongshin.primary] ([yongshin.type])
보조 용신: [yongshin.secondary] (있는 경우만)
희신: [yongshin.heeshin.join(', ')] (있는 경우만)
기신: [yongshin.gishin.join(', ')] (있는 경우만)
```

#### 5-2. 설명 문장
```
"용신은 사주 균형에 가장 필요한 오행으로, 사주에 따라 정해지는 고정된 오행입니다. 
당신의 용신은 [용신명]이며, 올해 세운에서 [용신명][조사] 들어오면 운세가 상승합니다."

희신이 있는 경우:
"희신은 용신을 돕는 오행으로, 올해 세운에서 [희신명들][조사] 들어오면 용신의 힘이 강해집니다."

기신이 있는 경우:
"기신은 용신을 방해하는 오행으로, 올해 세운에서 [기신명들][조사] 강하면 주의가 필요합니다."
```

**조사 처리**: `josa()` 함수 사용

## 6. 세부 운세 - 재물운 (money)

### 세운 기준
- **주도 십성**: 일간 대비 세운 지지(午)의 십성
- **보조 십성**: 일간 대비 세운 천간(丙)의 십성
- **목표 육친**: 재성(財星)
- **용신/기신**: 세운 오행이 용신/기신과 일치하는지
- **이벤트**: 충/합/형/파/해/없음

### 점수 계산식
```
재물운 점수 = 기본 3점
            + 세운 오행이 재성을 생하면 (+1)
            + 세운 오행이 재성이면 (+1)
            + 세운 오행이 재성을 극하면 (-1)
            + 용신과 일치하면 (+1)
            + 기신과 일치하면 (-1)
→ 1-5점으로 정규화 후 ×20 (100점 만점)
```

### 내용 생성 기준

#### 6-1. score
위 계산식으로 산출된 점수 (20~100점)

#### 6-2. summary
점수에 따른 요약:
- **4점 이상 (80점 이상)**: "재물 흐름이 좋은 해"
- **3점 이상 (60점 이상)**: "들어오지만 관리가 필요한 흐름"
- **그 외**: "지출 관리에 신경 써야 할 해"

#### 6-3. focus
점수에 따른 집중 포인트:
- **4점 이상**: "적극적인 투자"
- **3점 이상**: "지출 구조 개선"
- **그 외**: "안정적인 자산 관리"

#### 6-4. pros (좋은 점)
**조건 1**: 주도 십성이 식신 또는 상관인 경우
```
"나의 전문 기술이나 창의적인 아이디어가 시장에서 인정받으며 실질적인 수익으로 연결될 가능성이 매우 높습니다."
```

**조건 2**: 그 외
```
"[주도 십성 사용자 친화적 표현]로, 활동 범위가 넓어지며 새로운 수익원 창출에 유리한 흐름입니다."
```

#### 6-5. cons (주의점)
**조건 1**: 이벤트가 충 또는 형인 경우
```
"운의 기복이 심해지는 시기이므로, 벌어들이는 것만큼 나가는 지출(경쟁 비용, 충동 소비) 관리에 특별히 신경 써야 합니다."
```

**조건 2**: 그 외
```
"지엽적인 성과에 만족하여 큰 자산의 흐름을 놓치지 않도록 주의가 필요합니다."
```

#### 6-6. strategy (전략)
**조건 1**: 주도 십성에 '재'가 포함된 경우
```
"단기적인 시세 차익보다는 나의 '몸값'을 높이거나 장기적인 자산 가치에 집중하는 전략이 유효합니다."
```

**조건 2**: 그 외
```
"수익의 일정 부분을 반드시 안전 자산으로 묶어두어 운의 변동성에 대비하세요."
```

- **이벤트**: 충/합/형/파/해/없음

```
관계운 점수 = 기본 3점
            + 세운 오행이 해당 육친을 생하면 (+1)
            + 세운 오행이 해당 육친이면 (+1)
            + 세운 오행이 해당 육친을 극하면 (-1)
            + 용신과 일치하면 (+1)
            + 기신과 일치하면 (-1)
```

### 내용 생성 기준

#### 7-1. score

- **4점 이상 (80점 이상)**: "인연이 열리는 해"

- **그 외**: "관계에 신경 써야 할 해"

#### 7-3. focus
점수에 따른 집중 포인트:
- **4점 이상**: "새로운 만남"
- **3점 이상**: "능동적인 감정 전달"
- **그 외**: "소통 강화"

#### 7-4. pros (좋은 점)
**조건 1**: 이벤트가 합인 경우
```
"주변 사람들과의 깊은 유대감이 형성되며, 나를 지지해주는 든든한 아군이나 귀한 인연을 만날 수 있는 운입니다."
```

**조건 2**: 그 외
```
"자신감이 고취되면서 호감 있는 상대에게 본인의 매력을 자연스럽게 어필하기 좋은 시기입니다."
```

#### 7-5. cons (주의점)
**조건 1**: 이벤트가 충이고 궁위가 일 또는 월인 경우
```
"가까운 지인이나 배우자와의 소통 과정에서 예기치 못한 오해가 발생하거나 날카로운 언쟁이 생길 수 있으니 주의하세요."
```

**조건 2**: 그 외
```
"내 주장이 강해지는 해인 만큼, 타인의 조언을 간과하여 고립될 수 있는 점은 경계해야 합니다."
```

#### 7-6. strategy (전략)
고정 문장:
```
"상대방의 입장을 먼저 헤아리는 다정함이 곧 나의 복으로 돌아오는 해임을 잊지 마세요."

```

- 작년 기신해 → 올해 용신해: "기신 해에서 용신 해로 전환"
- 작년 일반 → 올해 용신해: "용신 해로 전환"

**조건 3**: 이벤트 변화

  - 작년 없음 → 올해 있음: `"[올해 이벤트]이/가 작용 추가"`
  - 작년 있음 → 올해 없음: `"[작년 이벤트]이/가 작용 해소"`

**조건 4**: 변화 이유가 없고 점수 차이가 5점 이상인 경우

## 참고사항

1. **모든 점수는 100점 만점으로 계산되지만, UI에서는 5점 만점(별점)으로 표시**
   - 변환식: `별점 = Math.round(점수 / 20)`

2. **조사 처리는 반드시 `josa()` 함수 사용**
   - 예: `josa(dominantTengod, '이/가')`

3. **사용자 친화적 표현은 `TENGOD_FRIENDLY_MAP` 사용**
   - 십성명 대신 설명문 사용

4. **이벤트 친화적 표현 매핑**:
   - 충 → 충돌
   - 합 → 협력
   - 형 → 압박
   - 파 → 분열
   - 해 → 해소
   - 없음 → 안정

5. **세운 상수 (2026년 기준)**:
   - 연도: 2026
   - 천간: 丙 (병)
   - 지지: 午 (오)
   - 작년: 2025년 을사년 (乙巳)
